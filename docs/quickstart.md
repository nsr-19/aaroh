## The Hint System

| Say | You Get | You DON'T Get |
|-----|---------|---------------|
| `hint` | One-sentence intuition, a question | Algorithm names, pattern names |
| `hint hint` | Narrower direction | Pattern name |
| `hint hint hint` | Pattern name + key insight | Code |

## The 6-Step DSA Flow

1. **Problem Setup** → Read, restate, clarify constraints
2. **Thinking / Stuck** → Use hint system if needed
3. **Dry Run** → Trace a test case by hand (BEFORE coding)
4. **Check-In** → Confirm approach + expected complexity
5. **Code** → Write solution, walkthrough, optimize
6. **Pattern Card** → Rating + structured card for SRS

## The 6-Step System Design Flow

1. **Problem Presentation** → One-liner, you drive from here
2. **Requirements** → Functional + non-functional + scope
3. **Estimation** → QPS, storage, bandwidth, cache
4. **High-Level Design** → Architecture components + data flow
5. **Deep Dive** → Go deep on 1–2 components
6. **Scaling & Trade-offs** → What breaks? What would you change?
